# 项目结果

本页集中记录项目里程碑、阶段性结果，以及适合对外介绍的总结口径。这里的目标不是展开全部实验细节，而是回答“项目已经做到什么”。

## 1. 里程碑

### 2025-10-29

完成 legacy 主体框架搭建并通过运行验证。

- 在 GeForce RTX 3060 上训练约 3 小时后，模型可以以 `81%-11%-8%` 的 `win-draw-lose` 结果战胜 `Random Agent`
- 这一阶段的重点是把完整规则、MCTS 和训练闭环跑通

### 2025-12-27

完成 v0 主体框架搭建并通过运行验证。

- 使用 C++ / CUDA 重写关键路径
- 在不降低训练效果的前提下，自博弈速度相比原 Python 功能性框架提升到约 `16x`
- 性能瓶颈从规则引擎 / 合法动作生成，进一步推向推理与调度链路

### 2026-02-26

v1 训练主线进入“强度迭代”阶段。

- `vs_random` 达到 `99.80%` 胜率
- 同一训练中，自博弈 `decisive_games` 从 `1.23%` 提升到 `81.72%`
- 80 模型锦标赛冠军为 `model_iter_032.pt`
- v1 在验证集中相对 v0 达到约 `25x-28x` 的速度提升

## 2. 简历口径摘要

以下内容用于概括项目方法与阶段成果：

- 实现规则引擎与合法动作生成器，将对弈流程拆解为原子阶段
- 搭建 AlphaZero 风格自博弈训练流水线
- 构建 `ChessNet`，使用三个策略头和一个价值头输出动作分布与局面价值
- 使用当前模型驱动的 MCTS 生成 `(s, π, z)` 风格训练样本
- 通过 C++ / CUDA 重写自博弈核心路径，并在工程上显著提升训练效率
- 搭建人机对弈前后端，支持玩家与训练后的模型直接对战

## 3. 对外介绍时可强调的能力

### 方法能力

- 从规则建模到训练闭环的端到端实现
- MCTS 与神经网络协同的强化学习训练方案
- 从 Python 功能性原型到 C++/CUDA 高性能路径的演进

### 工程能力

- staged 训练流水线
- 多 GPU 自博弈与训练调度
- 评估、锦标赛与 checkpoint 管理
- 前后端联通的人机对战系统

## 4. 当前未完成的问题

项目已经跨过“能否训练”的阶段，但仍在继续解决两个更难的问题：

- 棋力能否稳定随数据量和算力增长而提升
- 在较小算力预算下，模型能否稳定战胜更强的人类玩家

当前阶段的重点已不是单纯追求吞吐，而是提升“算力转化为棋力”的效率。
