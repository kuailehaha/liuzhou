# 六洲棋规则说明

## 1. 游戏概述

- 棋盘：`6 x 6` 网格，共 `36` 个交叉点
- 棋子：黑白双方各 `18` 枚
- 目标：通过提吃或相关移除机制使对方无子可用，从而获胜
- 平局保护：项目实现中加入总步数上限与无吃子判和规则，以避免训练与评估中的无限循环

## 2. 关键术语

### 方

由 `2 x 2` 的四枚同色、未被标记的棋子构成。

### 洲

由六枚同色、未被标记的棋子在同一行或同一列连续排成一线构成。

### 标记

仅在落子阶段触发。形成方可标记对方 `1` 子，形成洲可标记对方 `2` 子。

### 提吃

仅在走子阶段触发。形成方可提吃对方 `1` 子，形成洲可提吃对方 `2` 子。

## 3. 阶段结构

项目实现采用原子阶段语义。一个回合中的复合决策会被拆成多个阶段，以保证规则逻辑、动作生成和训练目标一致。

### `PLACEMENT`

- 当前玩家在空位落子
- 不能落在被对手标记的位置上
- 若新落子满足触发条件：
  - 形成方：生成 `1` 个待标记任务
  - 形成洲：生成 `2` 个待标记任务
- 若存在待标记任务，进入 `MARK_SELECTION`
- 若无待标记任务且棋盘已满，进入 `REMOVAL`
- 若无待标记任务且棋盘未满，交换玩家，继续 `PLACEMENT`

### `MARK_SELECTION`

- 当前玩家逐个处理待标记任务
- 合法目标是未被标记的对方棋子
- 若存在不在方或洲结构中的对方棋子，必须优先选择这些棋子
- 全部任务完成后：
  - 棋盘已满则进入 `REMOVAL`
  - 否则交换玩家并回到 `PLACEMENT`

### `REMOVAL`

- 若棋盘上存在被标记棋，则同时移除双方所有被标记棋
- 移除完成后进入 `MOVEMENT`，由 `WHITE` 先行
- 若棋盘填满但没有任何标记，则进入强制移除序列

### `FORCED_REMOVAL`

当棋盘填满但没有标记时：

1. `WHITE` 先移除一枚不属于方或洲结构的 `BLACK` 棋子
2. `BLACK` 再移除一枚不属于方或洲结构的 `WHITE` 棋子
3. 随后进入 `MOVEMENT`，由 `WHITE` 先行

### `MOVEMENT`

- 当前玩家移动自己的一枚棋子
- 仅允许沿上下左右移动一步到相邻空位
- 若移动后形成新结构：
  - 方：待提吃数 `+1`
  - 洲：待提吃数 `+2`
- 若存在待提吃任务，进入 `CAPTURE_SELECTION`
- 否则交换玩家，继续 `MOVEMENT`

#### 无可移动规则

如果轮到某方时该方没有合法移动，则该方可以通过特殊动作 `no_moves_remove` 先移除对手一枚棋子。随后进入 `COUNTER_REMOVAL`，由对手执行反移除。

### `CAPTURE_SELECTION`

- 当前玩家逐个执行待提吃任务
- 提吃目标的优先规则与标记阶段一致
- 每次提吃立即生效
- 若提吃后对方无子，则当前玩家立即获胜
- 否则当所有提吃完成后，交换玩家并返回 `MOVEMENT`

### `COUNTER_REMOVAL`

- 仅在执行 `no_moves_remove` 后触发
- 对手移除一枚行动方棋子，合法性限制与普通移除一致
- 反移除完成后，若局面未终结，则返回 `MOVEMENT`

## 4. 合法目标的优先级

标记、提吃、强制移除和反移除都遵循同一条核心原则：

- 若存在不在方或洲结构中的对方棋子，必须优先选择这些棋子
- 只有在不存在“普通棋”时，才允许选择结构内棋子

## 5. 终局与平局条件

### 胜利

以下情况判定当前行动方获胜：

- 提吃后对方无子
- 反移除后对方无子
- 其他规则驱动的移除后对方无子

### 平局

项目实现加入两条平局保护：

- 总步数达到 `144` 步仍未分胜负，判平
- 进入走子阶段后，连续 `36` 步未发生吃子，判平

其中第二条通过 `moves_since_capture` 计数器追踪；一旦棋子数量减少，计数器归零，否则持续累加。

## 6. 实现注意点

- 被标记棋在落子阶段不会立即移除，而是在后续 `REMOVAL` 阶段统一处理
- 被标记棋不能继续参与形成新的方或洲
- 每个 `GameState` 都会维护待标记数、待提吃数与无吃子计数
- Python 与 C++/CUDA 路径都需要保持同一套规则语义

## 7. 代码对应关系

当前规则实现与以下模块保持一致：

- `src/rule_engine.py`
- `src/move_generator.py`
- `src/game_state.py`
- `v0/src/rules/rule_engine.cpp`
- `v0/src/moves/move_generator.cpp`

规则变更需同步更新 Python 规则路径、`v0` C++ 实现、`v1` 训练目标语义，以及相关测试。
